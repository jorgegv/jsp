## Task 2: speed optimizations

Measure before/after with JNEXT (magic-port cycle counts) to confirm each change is worth keeping.
Benchmark harness: `tests/test_redraw_bench.c`, run with `make bench` (headless, prints
frame counts via the magic port). Phases: A0 = mark-dirty calibration, A = full-screen
redraw ×200, B = 5-sprite move+redraw ×1000.

Benchmark — before this session (`e609d42`) vs after (`707a473`):
- Full-screen redraw (A − A0): 2208 → 1702 frames for ×200  =>  **−22.9%** (11.04 → 8.51 frames/redraw).
- Sprite frame (B): 6554 → 6216 frames for ×1000  =>  **−5.2%** (6.55 → 6.22 frames/frame).
- A0 calibration unchanged (28 → 28), as expected.

- [x] Per-sprite compositing constants precomputed once per frame (`jsp_redraw_begin` → `jsp_frame_sprites[]`), so the per-cell path no longer recomputes them.
- [x] `jsp_redraw` DTT-walk hot loop rewritten in assembly (`jsp_redraw.asm`): foreground / uncovered cells are blitted inline from BTT, sprite-covered cells are handed to the C compositor `jsp_redraw_covered_cell`.
- [x] Earlier redraw fix: background-only cells draw straight from BTT (no per-cell scratch copy).
- x] Per-cell dispatch in the asm redraw loop tightened: the per-bit DTT mask is rotated instead of recomputed with a shift loop, and the cell index / column are running counters advanced once per bit instead of recomputed.
- [x] Assess the current C functions and apply the CLAUDE.md development recommendations regarding stack variables and `__z88dk_fastcall` calling convention. (fastcall on jsp_register/unregister_sprite, jsp_sprite_park/free, jsp_invalidate_rect; jsp_redraw_covered_cell scratch[8]+attr moved off-stack — main 12441→12408 B.)
- [x] Incremental screen addressing in `jsp_redraw`: the background-cell path derives the cell screen address from a constant 24-entry char-row table (`rd_rowtab[row] + col`) instead of calling `asm_zx_cxy2saddr`. Full-screen redraw 1730→1709 (−1.2%); sprite frame unchanged. A running-address-counter variant gave −6% on full redraw but +1.6% on sprite frames (per-cell upkeep taxes covered cells) — rejected.
- [x] Replace the per-cell `jsp_dtt_mark_dirty` calls in `mark_footprint_dirty` with a single "mark footprint rectangle" routine (`jsp_dtt_mark_rect`, clip applied by rectangle intersection). Sprite frame 6216→5910 (−4.9%); full-screen redraw unchanged.
