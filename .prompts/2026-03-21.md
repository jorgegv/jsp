## Task 2: speed optimizations

Done:
- Per-sprite compositing constants precomputed once per frame (`jsp_redraw_begin` → `jsp_frame_sprites[]`), so the per-cell path no longer recomputes them.
- `jsp_redraw` DTT-walk hot loop rewritten in assembly (`jsp_redraw.asm`): foreground / uncovered cells are blitted inline from BTT, sprite-covered cells are handed to the C compositor `jsp_redraw_covered_cell`.
- Earlier redraw fix: background-only cells draw straight from BTT (no per-cell scratch copy).

- Replace the per-cell `jsp_dtt_mark_dirty` calls in `mark_footprint_dirty` with a single "mark footprint rectangle" routine.
- Measure before/after with JNEXT (magic-port cycle counts) to confirm each change is worth keeping.
