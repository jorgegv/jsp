## Task 1: cleaner build (moved from 2026-06-04 Task 5)

- Make all builds generate the binary artifacts, maps, tap, dsk, etc. in a build/ directory instead of the repo root directory
- Fix the gitignore to just ignore the build/ directory and its contents
- Adapt the "clean" target to use that directory instead of the ever-growing list of files

### Notes (from a brief look on 2026-06-04, before this was deferred)

- Final outputs are controlled by zcc `-o`: main → main.tap/.map (root); CPC builds
  `-o CPC<NAME>` → `<NAME>.dsk` + named binary + `.map` (root); ZX test pattern
  rule outputs `tests/zx/<t>.bin`/`.tap`. These are the easy part — redirect `-o`
  into `build/`.
- HARDER: z88dk scatters per-source intermediates (`.o/.lst/.lis/.sym`) next to the
  sources (lib/, lib/cpc/, lib/zx/, tests/) — that's why `clean` has the
  `lib*/*.{map,lst,o,...}` wildcards. To make `.gitignore` *just* `build/`, those
  intermediates must also land in build/ (needs reworking the object paths /
  compile rules, or a single-invocation build with a build temp). Was about to
  test exactly what z88dk leaves behind when this was deferred.
- The "ever-growing list" pain in `clean` is the explicit CPC binary NAMES
  (CPCSPR2, CPCART, …); routing those `-o` outputs to build/ + `rm -rf build/`
  removes that list. The extension wildcards for scattered intermediates are
  stable (don't grow per-test).
