# Tasks

## Task 1: JSP for CPC - Design a Plan

- Goal: Adapt the JSP sprite library for the CPC target
- The document `doc/refs/CPC-SPRITE-ENGINE-ANALYSIS.md` gives context about the needs
- A sprite engine for Amstrad CPC targets that allow pixel-perfect sprite positioning
- Must work in Mode 0 (4bpp), Mode 1 (2bpp) and Mode 2 (1bpp)
- Must have the same architecture as for ZX. Relevant changes are the shifting tables and functions, and the compositing functions, which will be CPC- and Mode-specific.
- Must have compile-guards to compile in the desired mode
- There is only one ZX mode, but there may be more than 1 CPC compilation mode; at least:
  - CPC_MODE0: 1-pixel positioning in Mode 0
  - CPC_MODE1: 1-pixel positioning in Mode 1
  - CPC_MODE2: 1-pixel positioning in Mode 2
  - CPC_MODE1_MONO: 1-pixel positioning in Mode 1, but with 1-bpp assets and tiles
  - CPC_MODE0_FAST: 2-pixel positioning in Mode 0
  - CPC_MODE1_FAST: 4-pixel positioning in Mode 1
- Sprite coordinates can be real (i.e. 1-pixel resolution) in all JSP modes, JSP discards unused bits if needed
- Tests should be also adapted to CPC compilation modes
- All current JSP infrastructure (Makefile, tests, etc.) should be adapted to the newlibrary target
- You can design new sprite storage formats if needed, depending on the needs of each CPC mode.
- For the design of CPC testing, reuse/adapt/copy here the existing skill `caprice-testing` that exists in the project `~/src/spectrum/rage1/refactor_for_multiplatform/rage1/.claude/skills/`

- Create a document `doc/CPC-TARGET-PLAN.md` with a detailed design of the changes needed and a thorough implementation plan.
- Create a document `doc/cpc-target-tasklist.md` with the plan tasks as a checkbox list. Tasks in this file should be one-line, no exception. Description belongs in the plan document.
- Launch an independent and critic reviewer agent on the plan and task list documents and fix any findings
- Do not start coding
- Commit all the current and generated docs in branch `jsp_for_cpc`
